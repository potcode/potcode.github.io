移动端开发所踩过的坑(持续更新中…)

发表于 2017-02-06 |

摘要
本文会持续总结自己在移动开发中所踩过的坑，以及填坑的方法

科普一下hybrid app的定义吧

简称：混合模式移动应用，是介于web-app、native-app这两者之间的app,艰巨“Native App良好用户交互体验的优势”和“Web App”跨平台开发的优势”。

自然，各种不同版本的手机上，关于web的兼容性问题也表现的不同，不知不觉中，已经挖了无数的坑了。只能发现一个，填一个呗！

1. input [type=tel]

上面一段代码，为input标签定义了不可选择文本内容的属性。
造成的问题：Android 4.2.2 版本上，input绑定的所有事件全部失效。去掉这段代码，即恢复正常。

分析： user-select 这个属性或许android 4.2.2 不支持

2. input标签【部分android手机—虚拟键盘的backspace功能失效】

曾经遇到下面的情况，在一些android手机上，弹起的虚拟键盘中，点击“backspace”键，无法触发keyup事件。
有两种解决方法：
（1）android原生可以写一个监听“backspace”键的事件，这样在app中，就能正常的出发keyup事件。【不足之处：部分android手机的自带浏览器也会有这种问题，最常见的就是华为手机】
（2）写一个函数来监听input事件。通过触发焦点的时候，写一个循环函数，输入的值和上次保存的值作比较，值不相等，调用
输入键盘的方法，并把当前的值赋给上次保存的值。
input标签失去焦点的时候，循环停止。
